\# Project Rules



このプロジェクトはGitHub Pages上で動くブラウザ用シミュレータです。



\## 重要ルール

\- 1回の作業では1タスクだけ進める。

\- 大きな変更をする前に必ず計画を出す。

\- 実装後は必ず `npm run build` を実行する。

\- GitHub Pages用のbase path設定を壊さない。

\- 既存のUI/ロジックを壊さない。

\- 作業終了時に `progress.md` を更新する。



\## 完了条件

各タスクは以下を満たしたら完了：

1\. 変更内容が説明できる

2\. buildが通る

3\. 主要な画面または操作が確認済み

4\. progress.mdに記録済み



\## compact時に必ず残す情報

\- 現在の目的

\- 完了済みタスク

\- 変更したファイル一覧

\- 重要な設計判断

\- 未解決のバグ

\- 次にやるべきタスク

\- build/test結果
